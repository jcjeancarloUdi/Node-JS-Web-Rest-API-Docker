Conceito

O Docker é uma plataforma para desenvolvedores e administradores de sistemas desenvolverem, implementarem e executarem aplicativos com contêineres

Motivos para utilizar Docker:

  • Imagens Semelhantes em ambientes diferentes

    • Uma vez que sua aplicação seja transformada em uma imagem Docker, ela pode ser instanciada como container em qualquer ambiente que desejar

  • Aplicação ou Serviço é um Pacote Fechado, por isto o contêiner na ilustração

    • É possível empacotar sua aplicação e suas dependências, distribuir facilmente, diminui a documentação de infraestrutura.

  • Padronização na construção e versionamento de imagens

    • Através de arquivos de definição de imagens, pode se garantir o padrão, desenvolvedor pode escolher a versão da imagem que quer trabalhar de forma fácil e rápida.

  • Infraestrutura em código, mesma linguagem do desenvolvedor

    • Agora o cara da infraestrutura tem que saber programar.

História

Uma imagem é

um pacote executável que inclui tudo o que é necessário para executar um aplicativo - o código em tempo de execução, bibliotecas, variáveis ​​de ambiente e arquivos de configuração.

Um contêiner é

uma unidade padrão de software que empacota o código e todas as suas dependências para que o aplicativo seja executado de maneira rápida e confiável de um ambiente de computação para outro.
